## Тема курсовой

**Инструмент для кластеризации логов Apache**

---

## Задача проекта

Создать систему для хранения логов Apache (всех видов логов), поиска, группировки и обработки + применение различных методов кластеризации (с сохранением в БД данных каждой попытки кластеризации).

---

## Предполагаемый стек технологий

* **Python** - основная логика приложения и обработка логов;
* **FastAPI** - реализация REST API для загрузки, поиска и кластеризации логов;
* **MongoDB** - хранение логов и результатов кластеризации;
* **Docker** - контейнеризация приложения и упрощение развертывания.

---

## Видение проекта

Приложение с возможностью кластеризации логов Apache по одному из нескольких критериев: по endpoint, статус-коду, IP, а также раздельно для успешных запросов и ошибок (access/error). Для тестов будет реализован и использован генератор тестовых логов. Также будет предусмотрен модуль бэкапов и модуль статистики.

---

## Текущая реализация проекта

На текущем этапе настроена базовая инфраструктура проекта: подготовлено Docker-окружение, создан Python-сервис и реализовано подключение к MongoDB. Выполнена проверка взаимодействия приложения с базой данных - реализованы запись и чтение тестовых данных, а также настройка параметров подключения через переменные окружения.
